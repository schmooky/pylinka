| 2026-07-22 | §11.5 / hardening roadmap | Pre-1.0 review found the roadmap's "mobile hardening" item was still untouched: `grep` for `webglcontextlost` returned nothing across the whole repo. The only thing that existed was `onDeviceLost`, a single callback in the pixi runtime path that reported WebGPU device loss and did nothing about it. A backgrounded tab or a driver reset silently killed an effect on the backend the editor, the gallery and most games actually use. | Both WebGL2 backends now recover on their own. They listen for `webglcontextlost` and call `preventDefault()` (without it the browser never fires `webglcontextrestored` at all), go inert while the context is gone, and rebuild on the next `update()` after restore. Rebuild is deliberately deferred out of the event handler to the next step: a sub-emitter's VAOs reference its PARENT's buffers, and step order already guarantees the parent runs first, whereas event-listener order does not. The interpreted engine rebuilds in place via `createResources()`; the compiled handle re-runs its whole construction path and replays the knob log and emitter position onto the new sim, which is safer than trying to patch a 200-line constructor into a re-runnable shape. Particle state is not preserved: the buffers come back zeroed and the pool refills. WebGPU is deliberately NOT auto-recovered, because the device is shared per canvas and on the pixi path belongs to the renderer, so replacing it is the host's call; loss is detected, work stops, and both `onContextLost` and the older `onDeviceLost` fire. New surface: `contextLost` on both handles, `onContextLost` / `onContextRestored` options. **Verified in a real browser** by driving `WEBGL_lose_context` on the interaction lab: 60,572 alive → loseContext → stepping 30 frames throws nothing and `aliveCount()` reads 0 with `contextLost` true → restoreContext → 400 frames later 57,165 alive across all three systems, zero console errors. Not unit-tested: the path needs a real GL context, and a stub deep enough to fake one would test the stub. | — |
